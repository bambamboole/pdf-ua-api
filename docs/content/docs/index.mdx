## Three ways to create a PDF

Pick the entry point that matches what you already have — all three produce the same accessible
PDF/A-3a output.

<CardGrid>
  <Card title="From HTML" icon="document">
    Send a complete HTML document to `POST /convert`. Best when you already produce markup or use a
    templating engine. [HTML to PDF →](/html/html-to-pdf/)
  </Card>
  <Card title="From a URL" icon="external">
    Send a public URL to `POST /render/url` and the API fetches the HTML for you. Best for static
    sites or server-rendered pages. [URL to PDF →](/html/url-to-pdf/)
  </Card>
  <Card title="From a template" icon="list-format">
    Send a JSON template of rows and typed blocks to `POST /render/template`. Best for structured
    documents like invoices. [Templates →](/templates/structure/)
  </Card>
  <Card title="Validate & verify" icon="approve-check">
    Check any PDF against PDF/A-3a and PDF/UA-1 with `POST /validate`, or confirm a document came
    from this API. [Quick Start →](/getting-started/quick-start/)
  </Card>
</CardGrid>

## Run it in one command

The API ships as a single container — no databases, no headless browser, no external services.

```bash
docker run -p 8080:8080 ghcr.io/bambamboole/pdf-ua-api:latest
```

Then convert your first document:

```bash
curl -X POST http://localhost:8080/convert \
  -H "Content-Type: application/json" \
  -d '{"html":"<html><head><title>Document</title></head><body><h1>Hello World</h1></body></html>"}' \
  --output output.pdf
```
